# Olilo Influencer Platform — Setup Guide

**Who this is for:** Joon and Rich
**What this is:** Step-by-step instructions to get the influencer outreach system running
**Time to complete:** ~1.5–2 hours total
**Last updated:** 2026-05-15

## ARCHITECTURE (current)

```
Google Sheet (creator list)
        ↓ manual import (with DMs already written)
Airtable (Creators table)
        ↑↓ direct API
Vercel serverless functions (api/intake.js, api/apply.js, api/creators.js, etc.)
        ↑
Custom intake form at kit.olilosweet.com (web/index.html, deployed via Vercel)
```

Outreach copy (DMs + emails) is generated by **Alice** (system prompt at `agents/alice-prompt.md`) — either in a Claude Code session or via a one-off script that hits the Anthropic API. The output is written directly into Airtable.

**There is no n8n, no DigitalOcean droplet, no scheduled workflow runner.** Everything is either (a) a Vercel function triggered by a real user action, (b) a manual Airtable edit, or (c) a Claude Code session.

## SECTION 1: SET UP AIRTABLE

**When this is done:** A working CRM with all 5 tables, correct fields, and starter creators imported.

**Time:** ~45 minutes

### Step 1.1 — Create the base
1. Go to airtable.com and log in
2. Click "Add a base" → "Start from scratch"
3. Name the base: **Olilo Influencer CRM**

### Step 1.2 — Build the Creators table
1. Rename "Table 1" to "Creators"
2. Add all fields listed in `airtable/schema.md` → TABLE 1: CREATORS
3. For each field, use the correct field type
4. For single-select and multi-select fields, add all the options listed in the schema

### Step 1.3 — Build the other four tables
1. Click the "+" button next to the tables tab
2. Create "Outreach Log," "Shipments," "Posts," and "Applications"
3. Add all fields per the schema for each table

### Step 1.4 — Link the tables
In each of Outreach Log, Shipments, Posts, and Applications:
1. Find the "Creator" / "Linked Creator" field
2. Set it to link to: the **Creators** table

### Step 1.5 — Create the 7 views
1. In the Creators table, click "Add view" for each of the 7 views in the schema
2. Set up filters and groupings as described under each view

### Step 1.6 — Get your Airtable API credentials
1. Go to airtable.com/account → Developer Hub → Personal Access Tokens
2. Create a new token with scopes: `data.records:read`, `data.records:write`, `schema.bases:read`
3. Scope it to your "Olilo Influencer CRM" base only
4. Copy the token — save as **`AIRTABLE_TOKEN`**
5. Open your base URL: `airtable.com/[BASE_ID]/...` — copy the Base ID, save as **`AIRTABLE_BASE_ID`**
6. Open the Creators table and copy its table ID from the URL — save as **`AIRTABLE_TABLE_ID`**
7. Open the Applications table and copy its table ID — save as **`AIRTABLE_APPLICATIONS_TABLE_ID`**

**When this is done:** The branded intake form is live at `https://kit.olilosweet.com/`, posting submissions to `/api/intake` and writing to Airtable.

The form ships as part of this repo's Vercel deployment. See SECTION 3 (Vercel Deployment) below.

```
https://www.loom.com/embed/120d2a2af19e404fb34c216b935e60f6
```

Use an `<iframe>` near the top of the form. The video is the visual hook that the DM can't carry (Instagram doesn't preview Loom URLs).

When you generate per-creator intake links, append query params:

## SECTION 3: VERCEL DEPLOYMENT

**When this is done:** The intake form + endpoints are live and reachable at `kit.olilosweet.com`.

**Time:** ~15 minutes (if not already deployed)

### Step 3.1 — Set environment variables in Vercel
In the Vercel dashboard → Project Settings → Environment Variables, add:
- `AIRTABLE_TOKEN`
- `AIRTABLE_BASE_ID`
- `AIRTABLE_TABLE_ID`
- `AIRTABLE_APPLICATIONS_TABLE_ID`
- `RESEND_API_KEY` (for notification emails)
- `NOTIFY_EMAIL` (defaults to joon@adaptive.kitchen)

### Step 3.2 — Deploy
```bash
cd "Influencer Platform for Olilo"
vercel --prod
```

### Step 3.3 — Verify endpoints
- `/api/intake` (POST) — receives submissions from `web/index.html` (the kit.olilosweet.com form)

- `/api/update-creator` (PATCH, with admin token) — admin UI edits
- `/admin.html` — admin view
- `/kanban.html` — kanban view

## SECTION 4: OUTREACH COPY (DMs + Emails)

**When this is done:** You know how to generate and write outreach drafts into Airtable without n8n.

**Time:** ~5 minutes to read; ongoing usage

### The flow
1. **You** (in a Claude Code session, or via a one-off Python script in `scripts/`) point Alice at the rows that need drafts
2. **Alice** (`agents/alice-prompt.md`) returns one DM draft + one email draft per creator
3. **You** write them into the `DM Draft` and `Email Draft` fields via the Airtable API

### Generating drafts in a Claude Code session
- Open a Claude Code session in this project directory
- Give Claude the row IDs / creator names that need drafts
- Reference `agents/alice-prompt.md` as the system prompt for the agent
- Have Claude write the results back to Airtable via the Airtable API (creds in `.env.production`)

### Generating drafts via a one-off script
- Use the Anthropic Python SDK
- Read creator rows from Airtable
- For each row, call Claude with `alice-prompt.md` as the system prompt and the creator's fields as the user message
- Parse the response (Alice outputs `EMAIL DRAFT` then `--- DM DRAFT ---`) and PATCH the two fields back to Airtable

## SECTION 5: IMPORTING NEW CREATORS (Google Sheets → Airtable)

**This is the core ongoing operation.** Whenever Joon or Rich finds new creators in a Google Sheet, they get imported into Airtable. **DMs must be written at import time, not deferred.**

### Step 5.1 — Prepare the sheet
The Google Sheet should have at minimum these columns (matching the Creators table schema):
- Name
- Instagram Handle (with `@`)
- TikTok Handle (optional, with `@`)
- Audience Size
- Outreach Tier
- Category
- Why Olilo
- Assigned To
- List by

### Step 5.2 — Generate DMs before import
**Required.** Do not import rows that don't have DM drafts.

In a Claude Code session:
1. Paste or attach the Google Sheet (CSV export works fine)
2. Ask Claude to run each row through Alice (`agents/alice-prompt.md`) and produce `DM Draft` + `Email Draft`
3. Review the output — fix anything that misses the personalization or sounds off

### Step 5.3 — Import into Airtable
Two options:
- **CSV import via Airtable UI** — export the enriched sheet (with DM Draft + Email Draft columns) and use Airtable's CSV import to map columns
- **API import via script** — have Claude write the rows directly via the Airtable API (cleaner for larger imports, avoids manual mapping)

### Step 5.4 — Verify
After import, open the Pipeline (Kanban) view. Every new row should be in the "Not Contacted" column with `DM Draft` populated.

   - Copy the DM Draft text

   - Send via Instagram DM
   - Flip Status to "Contacted"
   - Set Last Contacted At to today

### Emails
1. Same flow, but copy Email Draft and send via Gmail (hello@olilosweet.com)
2. Set Status = "Contacted", log the touch, set Last Contacted At
